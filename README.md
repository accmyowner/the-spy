# Шпион онлайн

> **Что изменено в этом обновлении (только интерфейс/производительность):**
> - **Убрана нижняя панель статистики** («18 тем», «3–12 игроков»,
>   «в комнате», «0 ₽») — она больше не перекрывает кнопки.
> - **Поиск тем убран** (в этой сборке его и не было — подтверждено).
> - **Новая цветовая гамма** — вместо фиолетовой сделана «стальной циан +
>   янтарь + бирюза» на глубоком сине-чёрном фоне. Ни одного фиолетового
>   цвета не осталось.
> - **Новый AAA-фон** — только CSS и SVG, без видео: медленно движущиеся
>   градиенты, мягкий туман, светящиеся частицы (два слоя с параллаксом),
>   лёгкая сканлиния и виньетирование для читаемости текста. Вся анимация
>   идёт через transform/opacity (GPU), поэтому держит 60 fps; на телефонах
>   самые тяжёлые эффекты (blur, сканлиния, ближний слой частиц)
>   автоматически облегчаются, а при `prefers-reduced-motion` анимация
>   выключается полностью.
> - **Исправлено мигание.** Раньше каждый кадр таймера и каждое обновление
>   из Firebase перерисовывали всю страницу целиком, из-за чего фон
>   «моргал». Теперь фон и шапка строятся один раз в постоянную оболочку и
>   при перерисовках не пересоздаются — меняется только содержимое экрана.
>   Параллакс включается только на ПК (мышь), на телефоне не тратит ресурсы.
>
> **Онлайн-логика и Firebase не тронуты:** `firebase-config.js` и
> `index.html` побайтово идентичны, все функции комнат/ролей/голосования/
> таймеров на месте. Проверено: синтаксис, юнит-тесты подсчёта голосов и
> дымовой рендер всех экранов и состояний комнаты — без ошибок. Живую игру
> на нескольких устройствах прогоните на своём Firebase.

---

Игра «Шпион» с настоящими онлайн-комнатами: один создаёт комнату и получает
код, друзья вводят его на своих телефонах или компьютерах, роли раздаются
автоматически, таймер и раунды синхронизированы у всех в реальном времени.

Технически это обычный статический сайт (HTML + CSS + JS, без сборки и
npm), который использует **Firebase Realtime Database** как бесплатный
бэкенд для синхронизации между устройствами.

Файлы проекта:
```
index.html          — страница
style.css            — оформление (чёрная переливающаяся тема)
app.js                — вся логика игры и связь с Firebase
firebase-config.js  — сюда вы вставите свои ключи Firebase
README.md            — эта инструкция
```

## Часть 1. Создаём бесплатный проект Firebase

1. Откройте **https://console.firebase.google.com** и войдите с Google-аккаунтом.
2. Нажмите **«Add project» / «Создать проект»**.
3. Введите любое название (например, `spy-game`) → **Continue**.
4. На шаге Google Analytics можно выключить переключатель — для этой игры
   аналитика не нужна → **Create project**.
5. Дождитесь создания проекта → **Continue**.

## Часть 2. Включаем Realtime Database

1. В левом меню откройте **Build → Realtime Database**.
2. Нажмите **Create Database**.
3. Выберите регион (любой ближайший к вам, например `europe-west1`) → **Next**.
4. Выберите **Start in test mode** → **Enable**.
   (test mode даёт временный публичный доступ — в следующем шаге мы сразу
   поставим постоянные правила, так что это не проблема).
5. Откройте вкладку **Rules** этой базы данных, замените всё содержимое на:

   ```json
   {
     "rules": {
       "rooms": {
         "$roomCode": {
           ".read": true,
           ".write": true
         }
       }
     }
   }
   ```

6. Нажмите **Publish**.

   Это разрешает читать и писать только внутри узла `rooms/...` — именно
   там игра хранит комнаты. Всё остальное в базе по умолчанию закрыто.
   Правила намеренно простые и без входа в аккаунт (никакого логина у
   игроков нет) — это осознанный компромисс ради простоты: доступ к
   комнате имеет любой, кто знает её 5-символьный код. Для игры с
   друзьями этого достаточно.

## Часть 3. Получаем конфиг веб-приложения

1. В левом меню нажмите на шестерёнку рядом с **Project Overview** →
   **Project settings**.
2. Внизу раздела **Your apps** нажмите на иконку **`</>`** (Web).
3. Введите любое имя приложения (например, `spy-web`) → **Register app**.
   Firebase Hosting подключать не нужно, просто продолжите.
4. Появится объект `firebaseConfig` — он выглядит примерно так:

   ```js
   const firebaseConfig = {
     apiKey: "AIzaSyD...",
     authDomain: "spy-game-xxxxx.firebaseapp.com",
     databaseURL: "https://spy-game-xxxxx-default-rtdb.firebaseio.com",
     projectId: "spy-game-xxxxx",
     storageBucket: "spy-game-xxxxx.appspot.com",
     messagingSenderId: "123456789012",
     appId: "1:123456789012:web:abcdef1234567890"
   };
   ```

   Скопируйте эти значения — они понадобятся в следующем шаге.
   (Это публичный клиентский идентификатор проекта, а не секретный ключ —
   его можно без опасений публиковать в открытом репозитории; реальная
   защита данных обеспечивается правилами базы данных из Части 2.)

## Часть 4. Вставляем конфиг в проект

Откройте файл **`firebase-config.js`** в этом проекте и замените
плейсхолдеры своими настоящими значениями:

```js
export const firebaseConfig = {
  apiKey: "ваш реальный apiKey",
  authDomain: "ваш реальный authDomain",
  databaseURL: "ваш реальный databaseURL",
  projectId: "ваш реальный projectId",
  storageBucket: "ваш реальный storageBucket",
  messagingSenderId: "ваш реальный messagingSenderId",
  appId: "ваш реальный appId"
};
```

Сохраните файл. **`databaseURL` обязателен** — без него Realtime Database
работать не будет (в конфиге из консоли Firebase он уже есть, просто не
забудьте скопировать и его).

## Часть 5. Заливаем проект на GitHub

1. Создайте новый репозиторий на GitHub (например, `spy-online`) —
   публичный или приватный, не важно.
2. Загрузите в него все 4 файла проекта (`index.html`, `style.css`,
   `app.js`, `firebase-config.js` — уже с вашими ключами) в корень
   репозитория. Проще всего через веб-интерфейс: **Add file → Upload
   files** → перетащить все файлы → **Commit changes**.

## Часть 6. Включаем GitHub Pages

1. В репозитории откройте **Settings → Pages**.
2. В разделе **Build and deployment → Source** выберите **Deploy from a
   branch**.
3. **Branch**: `main` (или `master`), папка — **`/ (root)`** → **Save**.
4. Подождите 1–2 минуты, обновите страницу — вверху появится ссылка вида:

   ```
   https://ваш-логин.github.io/spy-online/
   ```

Откройте эту ссылку с телефона и с компьютера — игра запустится сразу,
без установки и регистрации.

## Как играть

1. Один игрок открывает сайт → **«Играть онлайн» → «Создать комнату»** →
   вводит имя → получает 5-символьный код.
2. Остальные открывают тот же сайт на своих устройствах → **«Играть
   онлайн» → «Войти по коду»** → вводят код и своё имя.
3. Хост выбирает тему и время на обсуждение, нажимает **«Раздать роли»**
   (нужно минимум 3 игрока).
4. У каждого на своём экране появляется его личная голографическая
   карточка — у всех, кроме одного, слово, у шпиона — только тема.
   Карточку можно открывать и закрывать сколько угодно раз.
5. Общий таймер обсуждения идёт синхронно у всех. Хост завершает раунд —
   все видят, кто был шпионом и какое было слово.
6. Хост запускает новый раунд той же темой или возвращается к выбору темы.

## Бесплатный лимит

Realtime Database на бесплатном тарифе Firebase (Spark) даёт 1 ГБ
хранилища и 10 ГБ трафика в месяц — для игр в компании друзей этого
хватит на очень долгое время, платёжная карта не требуется.

## Если что-то не работает

- **«Не получилось создать комнату» / «подключиться к комнате»** — почти
  всегда значит, что `firebase-config.js` не заполнен реальными
  значениями, или в Realtime Database не опубликованы правила из Части 2.
- **Комната не находится по коду** — проверьте, что код введён без
  пробелов и в том регистре, который показан (игра сама переводит буквы в
  верхний регистр).
- Если хост случайно закрыл вкладку — через несколько секунд хостом
  автоматически станет следующий по очереди игрок, никто не застрянет в
  комнате навсегда.
